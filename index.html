◇ What makes this different

RoBrain turns session context into structured, queryable decision records in Postgres: explicit rejected[], files_affected, embeddings for retrieval, lifecycle hooks, and a CLI that stays useful when notes grow long-lived or contradictory. It complements Claude Code Auto memory rather than replacing it.

Example capture → inject

Session: "let's use Zustand instead of Redux — Redux caused
re-render issues in the cart"

→ structured row: decision, rationale, rejected[], files_affected

Later: npx robrain inject --query "state management" --copy
→ pastes a one-line summary with date and confidence

Captured

  • Architectural decisions from agent sessions
  • Rationale and rejected alternatives
  • Files in scope, timestamps, confidence

Not captured

  • Your source code or file contents
  • Secrets, passwords, or personal data
  • Anything outside agent conversation turns

✦ What ships today

  • Passive capture — Sensing records turns; Perception extracts without the agent choosing what to remember.
  • rejected[] in Postgres — vetoes are first-class data, not prose buried in markdown.
  • Decision lifecycle — active vs superseded, linked history, robrain review to keep memory honest.
  • Team-shared store — one Postgres source of truth across machines and editors.
  • npx robrain explain <file> — file-scoped “why does this exist?” from stored decisions.
  • Conflict visibility — contradictions flagged; OSS resolves via review; cloud adds proactive warnings.

◎ Architecture (OSS)

On the developer machine: Sensing MCP watches sessions; the RoBrain CLI reviews, injects, and manages projects.

On your stack: Postgres (decisions + pgvector), Perception API (extract & store), and Synthesis — a batch job over the same DB (not on the hot session path): pnpm synthesis:run or npx robrain synth.

Planning API and Control MCP are Rory Plans cloud only; self-hosted OSS runs Postgres + Perception continuously and Synthesis on demand or on a schedule.

≋ Synthesis — three passes

Synthesis is the job that reads the whole decision corpus. The headline pass is the contradiction scan — e.g. “Use Bun” in February vs “Migrate back to Node” in May can surface as a direct reversal across separate sessions and vocabulary, with a conflicts_with edge and both rows flagged for review.

Drift detection clusters by topic and surfaces stance shifts without an explicit reversal (think REST → GraphQL across several decisions). Entity promotion turns recurring names like Stripe into compact planning_blocks with relationship history.

⌂ Self-hosted privacy

In self-hosted mode, conversation turns are processed by your local Perception API in Docker and stored in your Postgres — nothing is sent to Rory Plans. You’ll typically use ANTHROPIC_API_KEY (Haiku) plus an embedding provider key for vector search.

⚡ Quick start

One-time — robrain repo

pnpm install && pnpm build
pnpm docker:up
npx robrain install --self-hosted --repo-root "$(pwd)"

Per application repo

cd /path/to/your/project
npx robrain init-project

After init-project, Claude Code gets CLAUDE.md hooks for Sensing session start / turn / end — then capture is automatic. See the README for Cursor rules and troubleshooting.
